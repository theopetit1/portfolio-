Contexte Métier

Nous sommes une assurance (Branche Crédit & Épargne).

Le Problème :

  • Saturation des clients (trop d'appels).
  • Coût humain élevé des plateformes téléphonique.
  • Taux de transformation faible sur les appels "à froid".
Objectif : Passer d'un marketing de masse à un Marketing de Précision en utilisant la Data Science.

Méthodologie Technique

Notre approche pour transformer la donnée brute en score exploitable :

🧹
1. Nettoyage & Merge

• Fusion : Join sur `client_id` (Base Marketing + Base Finance).
• Filtres : Suppression des mineurs (<18 ans) et statuts incohérents.
• Imputation : Valeurs manquantes remplacées par la moyenne (Age) ou le mode (Geo).

📊
2. Discrétisation

• Transformation des variables continues (Age, IPC, Conso) en classes (Intervalles).
• Usage de pd.qcut() pour créer des quantiles équilibrés.

💻
3. Encodage "0-1"

• Objectif : Base 100% numérique.
• Fonction Python : pd.get_dummies(drop_first=True).
• Résultat : Chaque modalité devient une colonne binaire (0 ou 1).

⚖️
4. Sampling

• Gestion du déséquilibre (11% positifs).
• Random Oversampling : Duplication aléatoire des acheteurs pour équilibrer le Dataset d'entraînement (50/50).

Stratégie de Modélisation

Protocole de Test :

  • Split Data : 70% Entraînement / 30% Test (Validation).
  • Algorithme : Régression Logistique (Scikit-Learn).
  • Paramètres : max_iter=1000, solver 'lbfgs'.

Comparatif des Approches :

  • Standard : Biaisé (prédit trop de "Non").
  • Undersampling : Rapide mais perte d'info critique.
  • Oversampling (Retenu) : Meilleur compromis Stabilité / Performance.
